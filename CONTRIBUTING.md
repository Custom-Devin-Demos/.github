# Contributing to Allianz Open Source

Thank you for your interest in contributing to Allianz open source projects. We welcome contributions from the community and appreciate your efforts to improve our projects.

## Getting Started

Before you begin contributing, please take a moment to review our community guidelines:

- Read and follow our [Code of Conduct](CODE_OF_CONDUCT.md)
- Review our [Security Policy](SECURITY.md) for reporting vulnerabilities
- Familiarize yourself with the specific project's documentation and requirements

## Ways to Contribute

There are many ways to contribute to Allianz open source projects:

- **Report bugs**: If you find a bug, please create an issue with detailed information about how to reproduce it
- **Suggest enhancements**: We welcome ideas for new features or improvements to existing functionality
- **Submit code changes**: Fix bugs, implement features, or improve documentation through pull requests
- **Improve documentation**: Help us make our documentation clearer and more comprehensive
- **Review pull requests**: Provide constructive feedback on pending contributions

## Contribution Workflow

### 1. Fork and Clone

Fork the repository you want to contribute to and clone it to your local machine:

```bash
git clone https://github.com/YOUR_USERNAME/REPOSITORY_NAME.git
cd REPOSITORY_NAME
```

### 2. Create a Branch

Create a new branch for your contribution:

```bash
git checkout -b feature/your-feature-name
```

Use descriptive branch names that clearly indicate the purpose of your changes.

### 3. Make Your Changes

- Write clear, maintainable code that follows the project's coding standards
- Include tests for new functionality when applicable
- Update documentation to reflect your changes
- Ensure your code passes all existing tests
- Keep commits focused and write clear commit messages

### 4. Commit Your Changes

Write meaningful commit messages that explain what changes you made and why:

```bash
git add .
git commit -m "Brief description of your changes"
```

### 5. Push to Your Fork

```bash
git push origin feature/your-feature-name
```

### 6. Submit a Pull Request

Open a pull request from your fork to the main repository. In your pull request description:

- Clearly describe the problem you're solving or the feature you're adding
- Reference any related issues using GitHub's issue linking (e.g., "Fixes #123")
- Explain your approach and any design decisions you made
- List any breaking changes or dependencies introduced
- Include screenshots or examples if applicable

## Pull Request Process

1. **Review**: A maintainer will review your pull request and may request changes or provide feedback
2. **Discussion**: Be prepared to discuss your implementation and make revisions if needed
3. **Testing**: Ensure all automated tests pass and address any issues that arise
4. **Approval**: Once approved, a maintainer will merge your pull request
5. **Recognition**: Your contribution will be acknowledged in the project's release notes

## Code Review Guidelines

When reviewing pull requests, we consider:

- **Functionality**: Does the code work as intended and solve the stated problem?
- **Code quality**: Is the code readable, maintainable, and well-structured?
- **Testing**: Are there adequate tests to ensure the changes work correctly?
- **Documentation**: Is the documentation updated to reflect the changes?
- **Compatibility**: Do the changes maintain backward compatibility when appropriate?

## Reporting Issues

When reporting issues, please include:

- A clear and descriptive title
- Steps to reproduce the issue
- Expected behavior vs. actual behavior
- Your environment (OS, software versions, etc.)
- Any relevant logs, error messages, or screenshots

## Security Vulnerabilities

**Do not report security vulnerabilities through public GitHub issues.** Instead, please follow our [Security Policy](SECURITY.md) to report vulnerabilities responsibly.

## Code of Conduct

All contributors are expected to adhere to our [Code of Conduct](CODE_OF_CONDUCT.md). We are committed to providing a welcoming and inclusive environment for everyone.

Community leaders are responsible for enforcing these standards and will take appropriate action in response to any unacceptable behavior.

## Getting Help

If you have questions or need assistance:

- Check the project's documentation and README
- Search existing issues to see if your question has been answered
- Open a new issue with your question if you can't find an answer
- Contact the Allianz Open Source Program Office at [ospo@allianz.com](mailto:ospo@allianz.com)

## Recognition

We value all contributions to our projects. Contributors may be recognized through:

- Listing in the project's CONTRIBUTORS or AUTHORS file
- Acknowledgment in release notes
- Community recognition for significant contributions

Thank you for contributing to Allianz open source projects. Your participation helps make our software better for everyone!
